> 목적: 첨부된 7월 둘째 주 스케줄표가 `base_data/SCHEDULE_RULE.md`의 편성 규칙과

## 2. 일자별 검증

표기: **출근(n)** = 실제 출근 명단 / **자동off** = 플래너 정기휴무(표 미표기) / **신청off** = 휴무시트 표기분

### 7/6 (월) — 원장 Y·신 / 최소 9
- 출근(9): 성민·이은·윤정·은경·언경·지수·혜수·예진·서이
- 자동off(2): 미연·수현
- 판정: **정상**. 9명 = 하한 9 충족. 교정일 아님(정 미출근). 연차 섞임 양호(고·중·신규 혼재).

### 7/7 (화) — 원장 Y·오·신 / 최소 10
- 출근(10): 성민·이은·윤정·은경·언경·지수·혜수·미연·예진·수현
- 자동off(1): 서이
- 판정: **정상**. 10명 = 하한 10 충족.

### 7/8 (수) — 전체출근 + 야간시프트 / 최소 12 (※가용 11명)
- 야간시프트일은 "휴무자만 빼고 전원 출근" → 정기휴무 무시(`scheduleAssigner.ts:126~128`). 신청 휴무도 없어 **11명 전원 출근**.
- 주간(6): 성민·미연·서이·예진·은경·혜수
- 야간(5): 지수·수현·이은·윤정·언경
- **로직 재현 검증(정확 일치)**: `splitDayNightShift`(`scheduleAssigner.ts:37~48`)는
  야간고정(지수)을 야간에 먼저 넣고, 나머지를 `interleaveByCareer`(고→중→저→신규 순환)로
  정렬해 주/야 균형 배치한다. 직접 시뮬레이션 시 **주간=[성민,미연,서이,예진,은경,혜수],
  야간=[지수,수현,이은,윤정,언경]** 으로 표와 글자 단위까지 일치. → 산출이 코드대로임을 입증.
- 판정: **정상**. 야간고정 지수 → 야간 배치(규칙 25행) 충족. 12명 하한은 가용 인원이
  11명(민주 휴직)이라 물리적으로 불가 → 결함 아님(`min_staff_on_leave=11`과는 부합).

### 7/9 (목) — 원장 오·신(Y 휴무) / 최소 7 → ⚠️ **미달**
- 출근(5): 언경·혜수·미연·수현·서이
- 자동off(4): 성민·이은·윤정·은경 — **대표원장 핏 정렬**: Y가 목요일 휴무이므로
  `is_head_dentist_pick` 4명이 모두 목요일로 평일 휴무 정렬됨(`weeklyOffPlanner.ts:206~215, 238~244`).
- 신청off(2): 예진·지수(주차)
- 판정: **규칙 의도 vs 결과 충돌**.
  - 대표원장 핏 정렬(규칙 12행, "가중치 매우 높게")은 **충실히 동작**했다.
  - 그러나 코드상 핏 정렬은 평일 하한 검사를 **우회**한다(if 분기에서 floor 미확인,
    `weeklyOffPlanner.ts:238~244`). 핏 4명 + 신청 주차 2명이 한 요일에 겹쳐
    **출근 5명 < 하한 7명**. 야간 제외 평일 중 Y 휴무일이 하필 목(하한 7로 낮은 날)이라
    쏠림이 더 두드러졌다.
  - 하단에 예진·지수만 표기되고 성민·이은·윤정·은경이 안 보이는 것은 **정상**(자동 휴무 미표기 규칙).

### 7/10 (금) — 원장 Y·오·**정**(교정일) / 최소 9(without)·10(with)
- 출근(10): 성민·이은·윤정·은경·언경·지수·혜수·미연·수현·서이
- 신청off: 예진(주차) / 연차: 수진(?)
- 교정 인원 확인: 윤정·은경·언경·지수·혜수·미연 = **6명 ≥ 3명** → 교정 3명 고정 규칙 충족(규칙 27행).
  상한 없음(초과 교정 인원도 정상 출근, `scheduleAssigner.ts:141~157`).
- 판정: **정상**. 10명 충족(without 9·with 10 모두 만족).

### 7/11 (토) — 원장 Y·오·신 / 최소 7 → ⚠️ **미달**
- 출근(5): 이은·윤정·미연·수현·서이
- 자동off(2): 은경·지수(주말 휴무)
- 신청off: 성민·혜수·언경(주차) / 연차: 예진
- 신규 서이 토요일 출근 → 규칙 17행("신규는 토만 가능") 충족.
- 판정: **하한 미달(5 < 7)**. 단, **주말 휴무는 코드상 하한을 전혀 고려하지 않는다**
  (개인별 토/일 균등화 + 신청만 반영, `weeklyOffPlanner.ts:264~321`). 신청 주차 3 + 연차 1이
  토요일에 몰려 발생. 구현 설계상 "정상 동작"이나 설정 하한과는 불일치.

### 7/12 (일) — 원장 Y / 최소 7 → ⚠️ **미달(심함)**
- 출근(3): 성민·은경·지수
- 자동off(4): 이은·윤정·수현·서이
- 신청off: 혜수·언경·미연(주차) / 연차: 예진
- 팀장 출근 확인: **성민(팀장) 출근** → 규칙 14행("일요일 팀장 최소 1명") 충족
  (보장 로직 `weeklyOffPlanner.ts:323~343`).
- 신규 서이 일요일 미출근 → 규칙 17행 충족.
- 판정: **하한 미달(3 < 7)**. 토요일과 동일 원인 + 일요일 비번 인원이 겹쳐 더 심함.
  팀장·신규 규칙 자체는 충족.

## 3. 규칙별 종합 체크리스트

### 3-1. 충족 확인된 규칙 (근거 포함)

| 규칙 | 근거 | 결과 |
|---|---|---|
| 휴직자 제외 (민주·나야나) | 11명만 배정, 표 전체에서 미등장 | ✅ |
| 수요일 전체출근 + 야간 분리 | 8일 11명 전원, 주/야 split이 코드 재현과 글자 단위 일치 | ✅ |
| 야간고정(지수) → 야간 배치 | 8일 야간조에 지수 포함 | ✅ |
| 교정일 교정 인원 ≥3 (상한 없음) | 10일 교정 6명 출근 | ✅ |
| 신규(서이) 토만·일 불가 | 11일 출근 / 12일 미출근 | ✅ |
| 일요일 팀장 ≥1 | 12일 성민(팀장) 출근 | ✅ |
| 평일고정(언경) | 평일 전원 출근, 주말은 off/신청off | ✅ |
| 대표원장 핏 정렬 | 9일 Y 휴무에 성민·이은·윤정·은경 동반 휴무 | ✅(동작) |
| 하단 주차/연차 = 신청분만 | 자동휴무자 미표기, 신청자만 표기 | ✅ |
| 연차 섞임(저연차 편중 금지) | 각 출근일에 '고' 연차 1명 이상 포함 | ✅ |

### 3-2. 발견된 우려 사항 (의도 vs 결과)

1. **목·토·일 최소 인원 미달** (목 5<7, 토 5<7, 일 3<7)
   - **원인 A (목)**: 대표원장 핏 정렬이 평일 하한 검사를 **우회**(`weeklyOffPlanner.ts:238~244`)
     → Y 휴무일에 핏 4명이 무조건 몰림 + 신청 주차 2명 겹침.
   - **원인 B (토·일)**: 주말 휴무 배정에 **하한 개념이 아예 없음**(`weeklyOffPlanner.ts:264~321`).
     신청 휴무(주차·연차)가 주말에 몰리면 출근 인원이 곧바로 급감.
   - 판단: 코드는 "구현된 대로" 동작하므로 **버그라기보다 규칙·구현의 한계**.
     다만 운영상 하한 미달은 실사용에 영향이 크므로 보완 검토 권장.

2. **`min_staff` 컬럼 대부분 미반영**
   - 산출에 쓰이는 건 `min_staff_without_ortho`뿐(평일 최선노력 하한).
   - `min_staff_with_ortho`(교정일·주말 가중 하한)·`min_staff_on_leave`는 **산출 미사용**.
   - 결과적으로 교정일/주말의 더 높은 하한(예: 토·일 10)이 전혀 반영되지 않음.

### 3-3. 데이터 이상 (입력 점검 필요)

- **7/10(금) 연차 "수진"**: `staff` 명단에 **존재하지 않는 이름**(가장 유사한 '수현'은 같은 날 정상 출근).
  앱은 휴무시트의 이름을 직원 명단과 매칭 없이 그대로 표기하므로, **휴무신청서의 오타/명칭
  불일치** 가능성이 높다. 원본 휴무시트의 해당 행 확인 필요.

## 4. 결론

- **규칙 "준수형" 항목(전체출근·야간분리·교정 3명·신규·팀장·휴직제외·핏정렬·휴무표기)은
  모두 의도대로** 산출됐다. 특히 수요일 주/야 분리는 코드 시뮬레이션과 **완전 일치**해
  엔진이 설계대로 작동함을 입증한다.
- 반면 **요일별 최소 인원 보장은 목·토·일에서 미달**했다. 이는 ① 대표원장 핏 정렬의 하한 우회,
  ② 주말 휴무의 하한 미고려, ③ `min_staff_with_ortho`/`on_leave` 미반영이라는 **구현 한계**가
  신청 휴무 집중과 맞물려 나타난 결과다. 데이터 오류가 아니라 **로직 보완 과제**다.
- **즉시 조치 권장**: 7/10 "수진" 휴무 항목의 원본 확인(오타 여부).
- **개선 검토 권장**: 핏 정렬에도 하한 하한선 적용, 주말 휴무에 `min_staff` 반영,
  미사용 `min_staff` 컬럼 연결.

> `frontend/src/lib/weeklyOffPlanner.ts`, `frontend/src/lib/scheduleAssigner.ts`,
> DB `staff`·`schedule_setting`·`employee_type` (조회: SELECT only).
