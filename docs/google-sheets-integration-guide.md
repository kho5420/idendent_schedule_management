# Google 스프레드시트 연동 설정 가이드

이 문서는 "언제나이든치과 스케줄 관리" 앱에서 Google 스프레드시트를 **읽고 편집**할 수 있도록 연동하는 데 필요한 자격 증명(OAuth 클라이언트 ID)을 발급받는 방법을 설명합니다.

## 왜 필요한가요?

앱이 사용자 대신 Google 스프레드시트에 접근(읽기/쓰기)하려면, Google이 "이 앱은 신뢰할 수 있다"고 판단할 수 있는 ID가 있어야 합니다. 이를 **OAuth 2.0 클라이언트 ID**라고 하며, Google Cloud Console에서 무료로 발급받을 수 있습니다. (별도의 "API 키"는 필요하지 않습니다.)

## 1단계: Google Cloud 프로젝트 만들기

1. https://console.cloud.google.com 접속 후 Google 계정으로 로그인
2. 상단의 프로젝트 선택 메뉴 클릭 → "새 프로젝트"
3. 프로젝트 이름 입력 (예: `idendent-schedule`) → "만들기"
4. 생성된 프로젝트를 선택해서 들어갑니다

## 2단계: Google Sheets API 활성화

1. 왼쪽 메뉴에서 "API 및 서비스" → "라이브러리" 이동
2. 검색창에 "Google Sheets API" 입력 → 검색 결과 클릭
3. "사용 설정"(Enable) 버튼 클릭

## 3단계: OAuth 동의 화면 설정

1. "API 및 서비스" → "OAuth 동의 화면" 이동 → "대상" 메뉴 클릭 
2. User Type(사용자 유형)에서 "외부(External)" 선택 → 만들기
   - 외부를 선택하면, 앱을 정식 게시하기 전까지는 5단계에서 등록한 "테스트 사용자" 계정만 로그인할 수 있습니다
3. 앱 이름, 사용자 지원 이메일, 개발자 연락처 정보 입력 (필수 항목만 채우면 됩니다)
4. "범위(Scopes)" 단계는 비워두고 넘어가도 됩니다 (앱 코드에서 직접 권한을 요청합니다)
5. "테스트 사용자" 단계에서 실제로 로그인해서 사용할 본인의 Google 계정 이메일을 추가
6. 저장하고 완료

## 4단계: OAuth 클라이언트 ID 발급

1. "API 및 서비스" → "사용자 인증 정보" 이동
2. 상단 "+ 사용자 인증 정보 만들기" → "OAuth 클라이언트 ID" 선택
3. 애플리케이션 유형: **"웹 애플리케이션"** 선택
4. 이름 입력 (예: `schedule-app-web`, 자유롭게 지어도 무방)
5. "승인된 자바스크립트 원본"에 아래 주소 추가:
   - `https://kho5420.github.io` (배포된 GitHub Pages 주소)
6. "만들기" 클릭 → 클라이언트 ID가 발급됩니다 (`xxxxxxxxxxxx.apps.googleusercontent.com` 형태)

## 5단계: 발급받은 클라이언트 ID 등록 (개발자만)

- **로컬 개발**: `frontend/.env` 파일을 열어 `VITE_GOOGLE_CLIENT_ID=` 뒤에 발급받은 ID를 붙여넣기
- **배포 (GitHub Pages)**: GitHub 저장소 → Settings → Secrets and variables → Actions에 동일한 값을 등록 (배포 워크플로에서 `VITE_` 접두사를 붙여 빌드 시 주입합니다)

## 참고 사항

- **클라이언트 ID는 노출돼도 안전합니다.** API 키와 달리, 클라이언트 ID 자체는 프론트엔드 코드에 포함되어도 문제가 없습니다. 4단계에서 등록한 "승인된 자바스크립트 원본" 이외의 주소에서는 Google이 사용을 차단하기 때문입니다.
- **편집 권한 동의 안내**: 스프레드시트를 "편집"까지 하려면 로그인 시 더 넓은 권한(쓰기 권한)을 요청하게 됩니다. 처음 로그인할 때 Google 동의 화면에 "이 앱이 내 Google 스프레드시트를 보고 수정하도록 허용하시겠습니까?" 같은 안내가 표시되는데, 정상적인 절차이니 동의하면 됩니다.
- **"확인되지 않은 앱" 경고가 뜨는 경우**: 3단계 5번에서 본인 계정을 "테스트 사용자"로 등록했는지 확인하세요. 등록되지 않은 계정으로 로그인하면 경고가 표시될 수 있습니다.
- **이미 로그인한 적이 있다면**: 권한 범위(scope)가 변경되면 기존에 저장된 로그인 정보로는 편집이 되지 않을 수 있습니다. 이 경우 앱에서 로그아웃 후 다시 로그인해주세요.
