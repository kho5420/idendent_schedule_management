# 진료실 스케줄 관리 — UI 설계 스펙

**날짜:** 2026-06-04  
**범위:** 프론트엔드 UI (GitHub Pages, 백엔드 없음)  
**상태:** 확정

---

## 1. 개요

진료실 스텝의 월별 스케줄을 반자동으로 생성하는 웹 앱.  
사용자가 기존 스케줄 데이터(엑셀 또는 구글 스프레드시트)와 대상 월을 입력하면, 규칙에 따라 스케줄 초안을 생성하고 엑셀로 다운로드한다.  
모든 처리는 브라우저 내에서 실행되며 서버가 없다(GitHub Pages 정적 배포).

---

## 2. 사용자 흐름

```
[월 선택] → [입력 방식 선택: 엑셀 업로드 OR 구글 시트] → [스케줄 생성] → [미리보기 확인] → [엑셀 다운로드]
```

---

## 3. 화면 구성

### 3-1. 전체 레이아웃

단일 페이지 앱(SPA). 라우팅 없이 하나의 화면에서 모든 흐름을 처리한다.

```
┌─────────────────────────────────────┐
│  EDEN  진료실 스케줄 관리            │  ← 헤더
├─────────────────────────────────────┤
│  📅 대상 월 선택  [5월][6월][7월]…  │  ← 월 선택
├─────────────────────────────────────┤
│  입력 방식 선택                      │
│  ┌──────────────┐ ┌──────────────┐  │
│  │ 📁 엑셀 업로드│ │📊 구글 시트  │  │  ← 카드 선택형 (투트랙)
│  │  [드롭존]    │ │ [로그인 버튼]│  │
│  └──────────────┘ └──────────────┘  │
│  [⚡ N월 스케줄 생성]                │  ← 생성 버튼
├─────────────────────────────────────┤
│  2026년 N월 스케줄 미리보기  [⬇ 다운로드] │
│  ┌────┬──┬──┬──┬──┬──┬──┐         │
│  │날짜│월│화│수│목│금│토│         │  ← 미리보기 테이블
│  ├────┼──┼──┼──┼──┼──┼──┤         │
│  │1주 │…│…│…│…│…│…│         │
│  └────┴──┴──┴──┴──┴──┴──┘         │
└─────────────────────────────────────┘
```

### 3-2. 월 선택

- 현재 월 기준 앞뒤 3개월 칩(chip) 형태로 표시
- 기본 선택: 다음 달 (스케줄을 미리 짜는 용도)
- 선택된 칩은 퍼플 그라디언트로 하이라이트

### 3-3. 입력 방식 카드 (투트랙)

두 카드를 항상 나란히 표시. 한 번에 하나만 선택 가능.

**엑셀 업로드 카드**
- 드래그 앤 드롭 존 포함
- 클릭 시 파일 선택 다이얼로그 열림
- 허용 형식: `.xlsx`
- 파일 선택 후 카드에 파일명 표시
- 선택된 카드: 보라색 테두리 강조

**구글 스프레드시트 카드**
- "Google로 로그인" 버튼 표시
- 로그인 후: 스프레드시트 URL 입력 또는 목록에서 선택
- 인증 토큰은 `localStorage`에 저장 (재방문 시 자동 로그인 유지)
- Google Sheets API v4 사용, Client ID는 환경변수(빌드 타임 주입)로 관리

### 3-4. 생성 버튼

- 텍스트: `⚡ {N}월 스케줄 생성`
- 월 선택 + 입력 데이터 둘 다 준비된 경우에만 활성화
- 클릭 시 로딩 스피너 표시 (처리 중)

### 3-5. 미리보기 테이블

- 생성 완료 후 하단에 표시
- 컬럼: 주차 | 월 | 화 | 수 | 목 | 금 | 토
- 각 셀에 해당 날의 출근 스텝 이름 태그로 표시
- 수요일은 전체 출근이므로 "전체 출근" 텍스트로 표시
- 읽기 전용 (수정 불가)
- 가로 스크롤 지원 (모바일 대응)

### 3-6. 다운로드 버튼

- 미리보기 헤더 우측에 배치
- 클릭 시 `.xlsx` 파일 즉시 다운로드
- 파일명: `eden_schedule_YYYY_MM.xlsx`

## 4. 디자인 시스템

### 컬러
| 역할 | 값 |
|---|---|
| 배경 (앱 전체) | `#0f0f1a` |
| 카드 배경 | `#1e1e2e` |
| 보더 | `#2d2d44` |
| 액센트 (그라디언트) | `#6366f1` → `#8b5cf6` |
| 텍스트 기본 | `#e0e0f0` |
| 텍스트 서브 | `#888` |
| 성공 | `#22c55e` |

### 타이포그래피
- 폰트: 시스템 폰트 (`system-ui`, `Segoe UI`)
- 헤더: 26px bold
- 섹션 레이블: 12px uppercase, letter-spacing 1px
- 본문: 13–14px

### 컴포넌트 반경
- 카드: `border-radius: 12px`
- 버튼: `border-radius: 10px`
- 칩: `border-radius: 20px`
- 태그: `border-radius: 4px`

## 5. 기술 스택

| 항목 | 선택 |
|---|---|
| 프레임워크 | React 19 + TypeScript |
| 빌드 | Vite |
| 배포 | GitHub Pages |
| 스타일 | Tailwind CSS v4 |
| 엑셀 파싱/생성 | SheetJS (`xlsx`) |
| 구글 인증 | Google Identity Services (GIS) OAuth 2.0 |

## 7. 상태 관리

복잡한 전역 상태 라이브러리 없이 `App.tsx`의 `useState`로 관리한다.

```ts
type AppState = {
  selectedMonth: { year: number; month: number };
  inputMethod: 'excel' | 'google' | null;
  uploadedFile: File | null;          // 엑셀 업로드 시
  googleToken: string | null;         // 구글 로그인 시
  selectedSheetId: string | null;     // 구글 시트 ID
  scheduleData: ScheduleData | null;  // 파싱된 원본 데이터
  generatedSchedule: GeneratedSchedule | null; // 생성된 스케줄
  isGenerating: boolean;
};
```

## 8. Google OAuth 처리

- Google Cloud Console에서 OAuth 2.0 클라이언트 ID 발급
- 허용 도메인: GitHub Pages URL 등록
- Client ID는 Vite 환경변수(`VITE_GOOGLE_CLIENT_ID`)로 주입
- 토큰은 `localStorage('google_access_token')`에 저장
- 재방문 시 저장된 토큰으로 자동 복원, 만료 시 재인증
